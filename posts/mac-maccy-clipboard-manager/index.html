🧩为什么需要剪贴板工具

macOS 自带剪贴板一次只能保留当前内容。写代码、改配置、整理文档时，经常会在命令、路径、错误信息、链接、验证码和短文本之间反复复制。没有历史记录时，只要复制了新内容，前一个内容就被覆盖了。

Maccy 的价值在这里：它把最近复制过的内容保存成可搜索历史。需要旧内容时，用快捷键呼出，输入几个关键词，选中后直接用。它适合每天需要在多个窗口、终端、浏览器和编辑器之间来回搬运文本的人。

🍺用 Homebrew 安装

官方文档提供了 release 下载和 Homebrew 两种方式。日常维护更推荐 Homebrew，升级和卸载都更清晰。

brew install maccy
open -a Maccy

第一次启动后，Maccy 会出现在菜单栏。默认使用 Shift + Command + C 呼出历史窗口，也可以点击菜单栏图标打开。

⌨️基本使用方式

  1. 📋 像平时一样复制文本、链接或命令。
  2. ⌨️ 按 Shift + Command + C 打开 Maccy。
  3. 🔍 输入关键词搜索历史项。
  4. ✅ 回车或点击选择需要的内容。

如果你只是偶尔找回一段刚复制过的内容，默认行为已经够用。但如果想把它变成高频工作流，建议继续做下面这个设置。

⚡体验优化：开启自动粘贴

Maccy 默认偏保守，选中历史项时更像“把它重新放回剪贴板”。我更推荐打开自动粘贴：选中历史项后直接粘到当前光标所在应用，少一次 Command + V，也更符合“搜索后立即使用”的心智模型。

⚙️ Maccy 设置 🧭 行为 🚀 自动粘贴

打开路径是：菜单栏 Maccy 图标进入设置，找到 行为，开启 自动粘贴。这也是官方 FAQ 里解释“为什么选择历史项后没有粘贴”的关键设置之一。

🔐 如果开启后仍然不能自动粘贴，到 macOS 的 系统设置 > 隐私与安全性 > 辅助功能，确认 Maccy 已被允许控制电脑。自动粘贴需要这个权限，否则它无法把选中的历史内容发送到当前应用。

🎯适合哪些场景

  • 🧑‍💻终端里复制过命令、路径、环境变量，稍后又要找回来。
  • 📝写文档时频繁搬运链接、标题、短句和 Markdown 片段。
  • 🧪调试时需要在浏览器、编辑器、日志和 issue 页面之间切换。
  • 🧰想保留轻量工具，不想让剪贴板管理器变成复杂的资料库。

🔒几个使用建议

第一，剪贴板历史会涉及隐私，密码、token、密钥这类内容尽量交给密码管理器，不要依赖剪贴板长期保存。第二，Maccy 支持忽略复制事件和忽略特定 pasteboard 类型；如果你的工作流经常复制敏感内容，可以再去设置里收紧规则。

第三，记住一个快捷键就够了：Shift + Command + C。把它当成“打开复制历史”的入口，其他操作等真的需要时再学。
