macOS 下 Kaku + fish + Starship + Nerd Font + Catppuccin Powerline 配置

这篇文章基于我当前机器上的配置整理：Kaku 负责终端窗口和 shell 集成，fish 做默认 shell，Starship 生成 prompt，JetBrainsMono Nerd Font 和 Symbols Nerd Font 提供图标，Catppuccin Mocha 负责配色。

发布时间：2026-05-09 fish 4.7.0 Starship 1.25.1 Kaku 0.9.0
󰀵 fujikang~/Developer main 24 13:16

目标很简单：在 macOS 上用一套可复现、尽量少依赖插件管理器的方式，把终端配置成「打开 Kaku 就进入 fish，fish 自动加载 Starship，prompt 使用 Catppuccin Powerline，图标不乱码」的状态。下面的安装命令尽量使用 Homebrew。

当前环境快照

这篇文章参考了当前本机配置，关键版本如下：

  • Kaku：0.9.0，Homebrew cask 名称是 kakuku，来自 tw93/homebrew-tap。
  • fish：4.7.0。
  • Starship：1.25.1。
  • 字体：font-jetbrains-mono-nerd-font 和 font-symbols-only-nerd-font 已安装到 ~/Library/Fonts。
  • Homebrew 前缀：Apple Silicon 默认路径 /opt/homebrew。

1. 用 Homebrew 安装工具链

先安装 shell、prompt、Kaku 和 Nerd Font。Kaku 当前的 cask 名叫 kakuku，安装后应用名仍然是 Kaku。

brew install fish starship

brew tap tw93/tap
brew install --cask kakuku

brew install --cask font-jetbrains-mono-nerd-font
brew install --cask font-symbols-only-nerd-font

如果想把 fish 设置为默认登录 shell，先把 Homebrew fish 加入系统允许列表，再切换：

grep -qxF /opt/homebrew/bin/fish /etc/shells \
  || echo /opt/homebrew/bin/fish | sudo tee -a /etc/shells

chsh -s /opt/homebrew/bin/fish

2. 初始化 Kaku 的 fish 集成

Kaku 自带 shell 集成，负责把 Kaku 管理的 wrapper 加入 PATH、处理 TERM=kaku 在 ssh/sudo 下的兼容问题，并上报工作目录和语义 prompt 区间。初始化命令是：

kaku init
kaku doctor

当前机器上的 kaku doctor 是健康状态，重点信息包括：

  • SHELL 是 /opt/homebrew/bin/fish。
  • ~/.config/fish/conf.d/kaku.fish 存在，并会 source Kaku 管理的 fish 初始化文件。
  • ~/.config/kaku/fish/bin/kaku wrapper 可正常执行 kaku --version。

~/.config/fish/conf.d/kaku.fish 的入口保持很薄，具体逻辑交给 Kaku 管理文件：

# Kaku shell integration -- managed. Remove with: kaku reset
set -l _kaku_fish_init "$HOME/.config/kaku/fish/kaku.fish"
if test -f $_kaku_fish_init
    source $_kaku_fish_init
end

注意：~/.config/kaku/fish/kaku.fish 是 Kaku 管理文件，里面会写 PATH、ssh/sudo TERM fix、OSC 7、OSC 133、OSC 1337 和常用 abbreviation。不要手改这个文件；需要刷新时用 kaku init --update-only，需要移除时用 kaku reset。

3. 配置 Kaku 使用 fish

Kaku 的配置文件位于 ~/.config/kaku/kaku.lua。它兼容 WezTerm 的 Lua 配置 API，所以配置里会看到 local wezterm = require 'wezterm'。当前机器会先加载 Kaku 随应用附带的默认配置，再做少量覆盖：

config.font_size = 13
config.tab_bar_at_bottom = true
config.default_prog = { '/opt/homebrew/bin/fish', '-l' }
return config

如果你希望显式指定 Nerd Font，可以加上字体设置。字体名以 macOS 字体册里显示的名称为准：

config.font = wezterm.font('JetBrainsMono Nerd Font')
config.font_size = 13
config.default_prog = { '/opt/homebrew/bin/fish', '-l' }

如果 Powerline 分隔符或语言图标显示为方块，优先检查两件事：Kaku 当前 profile 使用的 font 是否是 Nerd Font，以及 font-symbols-only-nerd-font 是否已经安装。

4. 配置 fish 启动 Starship

当前 ~/.config/fish/config.fish 只保留了必要逻辑：交互式 shell 初始化 Starship，并把 Go bin 目录加入 PATH。

if status is-interactive
    starship init fish | source
end

fish_add_path $HOME/go/bin

Kaku 的 managed fish 文件里也会在检测到 starship 时初始化 prompt。实际维护时建议保证 Starship 初始化只由一个稳定入口负责：个人配置放 config.fish，Kaku 管理内容保持自动生成，不手动改。

5. Starship 的 Catppuccin Powerline 配置

Starship 配置位于 ~/.config/starship.toml。这套主题使用 Catppuccin Mocha 调色板，Powerline 左右分隔符是 、 和 ，模块顺序是 OS、用户名、目录、Git、语言运行时、Conda、时间、命令耗时和输入符号。

6. 验证

改完后建议按这个顺序验证：

fish --version
starship --version
kaku --version
kaku doctor

再打开 Kaku 新窗口，检查：

  1. echo $SHELL 是否是 /opt/homebrew/bin/fish。
  2. echo $PATH | string split ' ' | grep kaku 是否能看到 ~/.config/kaku/fish/bin。
  3. prompt 是否显示 macOS 图标、Powerline 分隔符、Git 分支和时间。
  4. 进入一个 Git 仓库、Node/Go/Python 项目，确认对应语言模块会出现。

常见问题

图标显示为方块

优先确认 Kaku 使用的是 Nerd Font。字体装了不代表终端一定在用它。建议选择 JetBrainsMono Nerd Font，并保留 Symbols Nerd Font 兜底。

ssh 到远端后提示 unknown terminal type kaku

Kaku 的 fish 集成里已经对 ssh 做了包装：当本地 TERM 是 kaku 时，用 TERM=xterm-256color 调用 ssh。不要手动删除 Kaku managed 文件里的这段逻辑。

sudo 下终端能力异常

同理，Kaku managed fish 文件也包装了 sudo，避免 root 环境缺失 TERMINFO_DIRS 后不认识 kaku。

Starship 初始化重复

当前配置中，Kaku managed 文件和个人 config.fish 都包含 Starship 初始化。一般不会阻塞使用，但长期维护时更推荐只保留一个明确入口。个人习惯是把自定义逻辑放在 ~/.config/fish/config.fish，Kaku 自动生成的部分只交给 kaku init 管。

小结

这套配置的核心边界是：Kaku 管窗口、字体和 shell 集成；fish 管启动入口；Starship 管 prompt；Catppuccin palette 管颜色；Nerd Font 管图标。边界清楚之后，问题定位会简单很多：乱码先看字体，prompt 不出现先看 fish 初始化，Kaku wrapper 不生效先跑 kaku doctor。
